# 应对"免费试稿"话术模组 v1.0

## 元信息

- **共识形成日期**：2026-02-02
- **最后更新**：2026-02-02
- **贡献者**：Creative-Career-OS 社区
- **适用场景**：客户要求先出方案再谈合作或付费

## 核心策略
通过**委婉拒绝**、**有偿试稿**、**比稿保护**等策略，避免"创意被白嫖"，同时不破坏客户关系。

## 详细内容

### 方案A：委婉拒绝法

**适用场景**：客户说"先做一稿看看，合适再付费"

**话术模板**：

**初级回应**（温和但坚定）：
```
感谢您的认可。关于"先试稿"，我想和您分享一下我的考虑：

设计是一个完整的创作过程，从需求分析到概念构思再到执行，需要投入大量时间和精力。如果先做方案，可能无法充分了解您的需求，反而影响最终效果。

我建议的流程是：
1. 先充分沟通需求，确保我理解您的期望
2. 签订合同并支付预付款后启动项目
3. 提供2-3个不同方向的初稿供选择
4. 根据反馈深化设计

这样既保证质量，也体现双方的诚意。您看是否可以这样安排？
```

**进阶回应**（专业且有理有据）：
```
理解您希望看到方案再决定的考虑。不过从专业角度，"先试稿"存在几个问题：

1. **需求不清**：没有充分调研就出手，方案可能偏离您的真实需求
2. **时间成本**：每个方案都需要2-3天的深度工作，免费试稿对创作者不公平
3. **版权风险**：方案一旦流出，即使未被采用，创意也可能被借鉴

我建议的替代方案：
- **方案A**：签订合作协议，支付30%预付款，我提供2-3个概念方向
- **方案B**：如确实需要先看方案，可以支付[X]元作为概念设计费，该费用可抵扣总项目款
- **方案C**：我可以提供过往类似项目的案例和思路，帮助您了解我的设计风格

您倾向于哪种方式？
```

### 方案B：有偿试稿法

**适用场景**：客户坚持要先看方案，但你不想完全拒绝

**话术模板**：
```
理解您的谨慎。为了平衡双方的需求，我建议采用"有偿试稿"的方式：

**试稿方案**：
- 费用：[X]元（可抵扣总项目款）
- 交付：1个概念方向 + 设计思路说明
- 版权：试稿阶段作品著作权归我，您获得审阅权
- 后续：如继续合作，该费用计入总款；如不合作，您不得使用试稿内容

这样既能让您看到我的设计能力，也保护了我的劳动成果。

您觉得这个方案可以接受吗？
```

### 方案C：比稿保护法

**适用场景**：客户明确表示要多家比稿

**话术模板**：
```
了解到贵司正在比稿。参与比稿前，我需要确认几个事项：

**比稿保护条款**：
1. **比稿费用**：参与比稿需要支付[X]元比稿费（中标后可抵扣，未中标不退还）
2. **方案数量**：提供[X]个概念方案
3. **版权保护**：未中标方案的著作权完全归我，贵司不得使用或借鉴
4. **保密协议**：双方签署保密协议，确保方案不被泄露给第三方
5. **反馈机制**：无论是否中标，希望获得评委的专业反馈

如果以上条款可以接受，我很乐意参与比稿。
```

## 使用指南

### 何时使用
- ✅ 客户要求"先试稿再谈合作"
- ✅ 客户说"出几个方案看看效果"
- ✅ 客户组织多家比稿但未提及费用
- ✅ 客户预算模糊但急于看方案

### 何时不适用
- ❌ 长期合作的老客户的小需求
- ❌ 已经签订合同并支付预付款的项目
- ❌ 客户明确表示"比稿费已包含在预算中"

### 注意事项
1. **态度要温和**：拒绝时要保持专业和礼貌，不要指责客户
2. **提供替代方案**：不要只说"不行"，要给出可行的替代建议
3. **强调价值**：让客户理解设计工作的专业性和价值
4. **准备退出**：如果客户坚持免费试稿，要有勇气放弃
5. **保留证据**：所有沟通记录要保存，以备纠纷

## 识别"白嫖"信号

以下信号表明客户可能想"白嫖"创意：
- ⚠️ 急于看方案，但对合同和付款含糊其辞
- ⚠️ 要求"先做一稿看看"，不提费用
- ⚠️ 比稿没有明确的评选标准和费用
- ⚠️ 客户内部"决策人"众多，意见难以统一
- ⚠️ 对设计流程和专业性缺乏基本尊重
