# 付款节点设置条款模组 v1.0

## 元信息
- **来源病例**：#002（如何从"按项目付款"谈判为"按阶段付款"）
- **共识形成日期**：2026-02-02
- **最后更新**：2026-02-02
- **贡献者**：Creative-Career-OS 社区
- **适用场景**：所有需要预付款和分期付款的创意项目

## 核心策略
通过**分阶段付款**和**里程碑确认**机制，降低回款风险，确保创作者在每个阶段都能获得相应报酬，避免"做完项目收不到钱"的困境。

## 详细内容

### 方案A：3331付款模式（推荐）

**核心思路**：将项目分为四个阶段，按3:3:3:1的比例付款，尾款作为质保金。

**付款节点设置**：
```
第X条 费用及支付方式

1. 项目总费用：人民币[X]元（大写：[X]元整）

2. 付款方式：分阶段支付

   第一阶段：合同签订后3个工作日内
   - 支付比例：30%
   - 支付金额：[X]元
   - 对应成果：项目启动、需求确认

   第二阶段：概念方案确认后3个工作日内
   - 支付比例：30%
   - 支付金额：[X]元
   - 对应成果：完成并提交概念设计方案

   第三阶段：深化设计确认后3个工作日内
   - 支付比例：30%
   - 支付金额：[X]元
   - 对应成果：完成并提交深化设计方案

   第四阶段：项目验收合格后7个工作日内
   - 支付比例：10%
   - 支付金额：[X]元
   - 对应成果：完成全部交付物，项目验收合格

3. 付款方式：银行转账
   - 户名：[乙方账户名]
   - 开户行：[开户银行]
   - 账号：[银行账号]

4. 发票开具：乙方在收到每笔款项后[X]个工作日内开具等额增值税发票

5. 逾期付款：
   - 甲方逾期付款的，每逾期一日按应付款项的万分之五支付违约金
   - 逾期超过15日的，乙方有权暂停工作直至收到款项
   - 逾期超过30日的，乙方有权解除合同并要求甲方支付已完成工作的费用
```

### 方案B：预付款+验收付款模式

**核心思路**：适用于周期较短或金额较小的项目，简化付款流程。

**付款节点设置**：
```
第X条 费用及支付方式

1. 项目总费用：人民币[X]元（大写：[X]元整）

2. 付款方式：

   预付款：合同签订后3个工作日内支付50%，即[X]元
   - 项目启动的必要条件
   - 收到预付款后开始计算项目周期

   验收款：项目验收合格后5个工作日内支付50%，即[X]元
   - 甲方确认终稿并签署验收单后支付
   - 乙方收到尾款后交付全部源文件

3. 验收标准：
   - 乙方提交的设计方案符合合同约定的需求
   - 甲方在收到终稿后[X]个工作日内完成验收
   - 超过[X]个工作日未提出异议的，视为验收合格

4. 修改与验收：
   - 合同约定的修改次数内，乙方免费修改
   - 超出约定修改次数的，甲方需支付额外费用
   - 因甲方原因导致项目延期的，付款时间相应顺延
```

### 方案C：月结模式（长期合作）

**核心思路**：适用于长期合作或按月服务的项目。

**付款节点设置**：
```
第X条 费用及支付方式（月结模式）

1. 服务费用：
   - 月度服务费：人民币[X]元/月
   - 或按项目计费，单价为[X]元/项目

2. 结算周期：
   - 每月[X]日统计上月工作量
   - 每月[X]日前乙方提交对账单和发票
   - 甲方在收到发票后[X]个工作日内付款

3. 预付款（可选）：
   - 合作开始前支付首月服务费作为预付款
   - 或支付[X]元作为合作保证金，合作结束后退还

4. 年度结算（可选）：
   - 年度累计消费满[X]元，享受[X]折优惠
   - 或年度结束后根据总消费额给予返点

5. 付款保障：
   - 连续两个月未按时付款的，乙方有权暂停服务
   - 累计欠款超过[X]元的，乙方有权解除合同
```

## 使用指南

### 何时使用
- ✅ 金额较大的项目（建议超过[X]元）
- ✅ 周期较长的项目（超过2周）
- ✅ 新客户或信用未建立的客户
- ✅ 需要垫付成本的项目（如摄影、印刷等）

### 何时不适用
- ❌ 金额很小的项目（如几百元的简单设计）
- ❌ 长期合作且信用良好的老客户
- ❌ 客户明确要求验收后一次性付款且无法协商的

### 注意事项
1. **明确付款条件**：每个阶段的付款前提条件要写清楚
2. **设置验收期限**：避免客户无限期拖延验收
3. **保留工作证据**：保存每个阶段的交付记录
4. **及时催款**：到期未付款要及时提醒
5. **准备退出机制**：明确什么情况下可以暂停或终止项目

## 话术模板

### 面对"验收后一次性付款"要求时

**初级回应**：
```
感谢您的信任。关于付款方式，我建议采用分阶段付款的模式。

这样对我们双方都更有保障：
- 您可以分阶段确认成果，降低风险
- 我可以更专注地投入项目，确保质量

我建议的付款安排是：
- 签约时支付30%启动项目
- 概念方案确认后支付30%
- 深化设计确认后支付30%
- 验收后支付尾款10%

这样既符合行业惯例，也能确保项目顺利推进。您看这样的安排是否可以？
```

**专业回应**：
```
理解贵司的财务流程。从项目管理角度，我建议采用里程碑付款的方式。

原因如下：
1. 风险控制：分阶段确认成果，降低双方风险
2. 现金流：我需要支付团队成本和外包费用，预付款是必要的
3. 项目保障：确保我能全身心投入，不被资金问题分心

我建议的方案：
- 预付款30%：项目启动
- 中期款40%：主要成果交付
- 尾款30%：验收完成

或者，如果贵司确实需要验收后付款，我需要：
1. 提高报价10-15%以覆盖资金成本
2. 签订正式合同并加盖公章
3. 验收期限明确为[X]个工作日

您看哪种方案更合适？
```

**应对拖欠付款时**：
```
您好，根据合同约定，[阶段名称]的付款已于[日期]到期，目前还未收到款项。

为确保项目顺利推进，请您尽快安排付款。

付款信息：
- 金额：[X]元
- 账户：[账户信息]

收到款项后，我将立即继续[下一步工作]。

如有特殊情况，请提前告知，我们可以协商解决。
```
