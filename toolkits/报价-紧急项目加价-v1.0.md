## 核心概念

### 为什么紧急项目要加价？

**成本增加**：
- 需要加班，影响工作生活平衡
- 需要推迟或暂停其他项目
- 需要外包部分工作
- 压力增大，风险提高

**机会成本**：
- 占用原本可以接其他项目的时间
- 影响现有项目的质量
- 牺牲休息和学习时间

**价值体现**：
- 快速响应本身有价值
- 为客户解决燃眉之急
- 专业能力的溢价

## 紧急程度分级

### 普通加急（1.2-1.5倍）
**特征**：
- 时间比正常压缩20-30%
- 不影响其他项目
- 不需要加班

**示例**：
- 正常2周的项目，要求10-12天完成
- 可以调整工作节奏完成

### 高度加急（1.5-2倍）
**特征**：
- 时间压缩50%左右
- 需要加班或调整其他项目
- 有一定压力但可控

**示例**：
- 正常2周的项目，要求7天完成
- 需要晚上和周末工作

### 极限加急（2-3倍）
**特征**：
- 时间压缩70%以上
- 必须全力投入，暂停其他所有工作
- 高压力，高风险

**示例**：
- 正常2周的项目，要求3-5天完成
- 需要通宵或连续高强度工作

## 加价策略

### 策略1：阶梯式加价

```
正常周期：100%
压缩20%：120%
压缩30%：130%
压缩50%：150%
压缩70%：200%
压缩80%：250-300%
```

**示例**：
```
品牌设计项目，正常报价5万，周期2周

客户要求：
- 10天完成（压缩30%）：6.5万
- 7天完成（压缩50%）：7.5万
- 5天完成（压缩65%）：10万
- 3天完成（压缩80%）：12-15万
```

### 策略2：条件式报价

**话术模板**：
```
理解您的紧急需求。为了确保质量，我们有以下方案：

方案A：标准时间（10天）- 5万
- 正常排期
- 保证质量

方案B：加急（7天）- 7.5万（1.5倍）
- 需要调整其他项目
- 我们会加班完成

方案C：极限加急（3天）- 12万（2.4倍）
- 需要暂停其他所有项目
- 团队全力投入
- 包含2轮修改

您看哪个方案更适合？或者我们可以协商一个中间方案？
```

### 策略3：成本透明法

**话术模板**：
```
紧急项目需要额外成本：

1. 人力成本：
   - 需要2位设计师同时工作（正常1位）
   - 需要3天通宵加班
   - 加班费：X元

2. 机会成本：
   - 需要推迟另一个项目
   - 损失收入：X元

3. 风险成本：
   - 时间紧张，质量风险增加
   - 需要增加校对和审核

因此，紧急项目报价：X万（正常价格的X倍）
```

## 沟通话术

### 客户说"能不能快点，不加价"

**错误回应**：
- "好的，我尽量"
- "不加价可能来不及"

**正确回应**：
```
"理解您的紧急需求。让我评估一下：

如果按正常时间，我们可以保证最佳质量。
如果加急，我们需要：
- 加班工作
- 调整其他项目安排
- 投入更多资源

这些都会产生额外成本。

不过，考虑到长期合作，我们可以：
方案1：压缩20%时间，加10%费用
方案2：保持原价，但质量可能受影响
方案3：分阶段交付，先做最核心的部分

您倾向哪个方案？"
```

---

### 客户质疑"为什么加急要加这么多钱"

**回应**：
```
"这是一个好问题。让我解释一下：

正常2周的项目，我们需要：
- 前期调研：3天
- 概念设计：4天
- 深化设计：4天
- 修改完善：3天

如果压缩到5天：
- 调研不能省，但其他环节必须大幅压缩
- 需要2-3人同时工作（正常1人）
- 需要连续加班
- 需要暂停其他项目

这就像：
- 普通快递 vs 次日达
- 经济舱 vs 商务舱
- 都是同样的服务，但紧急程度不同，成本不同

我们加价的部分，就是覆盖这些额外成本。
当然，如果时间上可以灵活，我们可以按正常价格。"
```

---

### 客户说"别家不加价也能做"

**回应**：
```
"每家公司的运作方式不同。

可能的情况：
1. 他们正好有空档期
2. 他们团队规模大，可以灵活调配
3. 他们的报价本身就包含了弹性空间

我们的情况是：
- 目前排期已满
- 接您的加急项目，需要调整其他安排
- 所以需要加急费用

如果您不急，可以等我们的正常排期，按原价。
如果很急，加急费是合理的成本体现。

您可以根据实际情况选择。"
```

## 判断是否应该接紧急项目

### 评估清单

**项目本身**：
- [ ] 预算是否足够（加急费后）
- [ ] 客户是否值得（长期价值）
- [ ] 项目是否有意义（作品质量）

**自身状况**：
- [ ] 身体和精神状态是否允许
- [ ] 其他项目是否可以调整
- [ ] 是否有足够资源（人手、时间）

**风险评估**：
- [ ] 时间太紧是否影响质量
- [ ] 失败的风险有多大
- [ ] 对声誉的影响

### 决策矩阵

| | 预算充足 | 预算紧张 |
|:---|:---|:---|
| **重要客户** | ✅ 接，全力做 | ⚠️ 协商，适当让步 |
| **普通客户** | ✅ 接，正常做 | ❌ 不接或延期 |

## 紧急项目管理

### 接项目前

**明确约定**：
- 交付时间和里程碑
- 修改次数（建议减少，如1-2轮）
- 付款方式（建议提高预付款比例）
- 质量标准（双方认可）

**合同条款**：
```
本项目为加急项目，交付时间为[X]日。
客户确认因时间紧急，接受以下调整：
1. 修改次数限制为[X]轮
2. 部分细节可在交付后优化
3. 加急费用为合同总额的[X]%
4. 如客户原因导致延期，加急费不退
```

### 项目进行中

**每日同步**：
- 每天向客户汇报进度
- 及时确认方向，避免返工
- 有问题立即沟通

**风险控制**：
- 先做核心部分
- 保留修改时间
- 准备Plan B

### 项目交付后

**总结复盘**：
- 记录实际成本
- 评估是否盈利
- 总结经验教训

## 常见误区

### 误区1：不加价接急单
**症状**：为了维护关系，免费加急
**后果**：自己吃亏，客户不珍惜
**解决**：明确加急费是合理成本

### 误区2：加价太高吓跑客户
**症状**：3倍、5倍加价
**后果**：失去项目
**解决**：合理加价，1.5-2倍为宜

### 误区3：接了做不完
**症状**：为了钱硬接，最后延期
**后果**：损失信誉，可能赔偿
**解决**：评估能力，做不完不接

### 误区4：不加筛选接所有急单
**症状**：什么急单都接
**后果**： burnout，质量下降
**解决**：选择性接单
