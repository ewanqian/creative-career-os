# CCOS-Agent Prompt 使用指南

> 本指南教你如何与任何大语言模型（ChatGPT、Claude、文心一言等）对话，快速生成符合社区标准的真实故事文档。

---

## 什么是 CCOS-Agent？

**CCOS-Agent** 不是一个具体的软件，而是一种**与 AI 对话的方法论**。通过预设的 Prompt（提示词），引导 AI 帮助你：

1. **结构化回忆**：一步步梳理你的经历
2. **自动脱敏**：识别并替换敏感信息
3. **格式生成**：输出符合标准的 Markdown 文档

**核心原则**：AI 是辅助工具，最终审查和脱敏确认必须由你完成。

---

## 快速开始（3 分钟版）

### Step 1：复制基础 Prompt

```
你是一位专业的创意行业案例整理助手。请帮助我整理一个设计/创意项目的真实经历，用于开源社区分享。

【任务目标】
将我的项目经历转化为结构化的"真实故事"文档，要求：
1. 完全脱敏（替换所有可识别信息）
2. 结构清晰（背景-困境-应对-反思）
3. 建设性（聚焦于"如何解决"）

【输出格式】
请按照以下结构输出：

## 真实故事标题：[一句话概括核心问题]

### 基本信息
- **涉及领域**：[合同/报价/沟通/版权/其他]
- **职业身份**：[自由设计师/插画师/新媒体运营/其他]
- **项目类型**：[品牌设计/UI设计/插画/视频/其他]

### 事件背景
[描述项目的来龙去脉]

### 核心困境
[清晰描述遇到的具体问题]

### 你的应对
[描述你当时采取的行动]

### 反思与教训
[总结经验教训]

【脱敏规则】
必须替换以下信息：
- 公司名 → [某大型公司]/[某知名平台]
- 人名 → [客户A]/[对接人B]
- 项目名 → [某品牌升级项目]
- 金额 → [约五位数]/[约六位数]
- 时间 → [去年春季]/[约半年前]
- 地点 → [一线城市]/[某商业中心]

现在，请告诉我：你遇到了什么项目困境？
```

### Step 2：粘贴到 AI 对话窗口

- ChatGPT: [chat.openai.com](https://chat.openai.com)
- Claude: [claude.ai](https://claude.ai)
- 文心一言: [yiyan.baidu.com](https://yiyan.baidu.com)

### Step 3：回答 AI 的提问

AI 会一步步问你：
1. "这是什么类型的项目？"
2. "遇到了什么具体问题？"
3. "你当时怎么应对的？"
4. "最后的结果如何？"

### Step 4：审查和修改

**必须人工检查**：
- [ ] 所有敏感信息都已替换
- [ ] 时间、地点、金额已模糊化
- [ ] 没有透露可识别的细节
- [ ] 内容聚焦于解决方案

### Step 5：提交到 GitHub

复制最终文档，在 [Issues](../../issues) 区提交。

## 进阶使用（完整版 Prompt）

如果你希望获得更高质量的输出，使用这个完整版：

```
你是一位资深的创意行业案例整理专家，也是开源社区 Creative-Career-OS 的贡献者。请帮助我整理一个真实的项目经历，转化为高质量的"真实故事"文档。

【背景知识】
Creative-Career-OS 是一个开源的创意从业者生存指南项目，通过"真实故事-共识-模组"的飞轮机制，将个人经验转化为公共资产。

- "真实故事"：匿名脱敏的真实困境描述
- "共识"：社区讨论形成的解决方案
- "模组"：结构化的可复用工具

【你的任务】
通过对话引导我完成以下步骤：

步骤 1：项目背景梳理
- 这是什么类型的项目？
- 如何接触到这个项目的？
- 客户的基本情况（行业、规模）？
- 项目的初步范围和预期？

步骤 2：困境识别
- 具体遇到了什么问题？
- 问题是什么时候发现的？
- 造成了什么影响（时间/金钱/心理）？
- 当时的第一反应是什么？

步骤 3：应对过程
- 尝试了哪些解决方案？
- 与客户的沟通是怎样的？
- 遇到了什么阻力？
- 最终结果如何？

步骤 4：反思总结
- 如果重来，你会怎么做？
- 哪些信号应该提前警惕？
- 有什么预防建议？
- 希望获得社区什么帮助？

【输出要求】
1. **完全脱敏**：
   - 公司名 → [某大型互联网公司]/[某知名电商平台]/[某新消费品牌]
   - 人名 → [客户A]/[对接人B]/[总监C]
   - 项目名 → [某品牌升级项目]/[某APP改版项目]
   - 金额 → [约五位数预算]/[约六位数]/[约中四位数]
   - 时间 → [去年春季]/[约半年前]/[2024年初]
   - 地点 → [一线城市]/[某商业中心]/[某创意园区]
   - 作品名 → [某系列海报]/[某主题创作]

2. **结构完整**：
   - 真实故事标题：一句话概括核心问题
   - 基本信息：领域、身份、项目类型
   - 事件背景：来龙去脉
   - 核心困境：具体问题
   - 你的应对：行动过程
   - 反思与教训：经验总结

3. **建设性**：
   - 聚焦于"如何解决"
   - 提供可操作的建议
   - 避免纯情绪宣泄

【对话开始】
请问我："你好！我很乐意帮你整理这个真实故事。首先，请简单描述一下这是什么类型的项目？"

然后基于我的回答，继续提问，直到收集到完整信息。
```

## 使用技巧

### 技巧 1：分段输入

如果你的经历很复杂，可以分段告诉 AI：

```
我先描述背景：[背景描述]

接下来是困境：[困境描述]

然后是应对：[应对描述]

最后是反思：[反思描述]

请帮我整理成完整的真实故事格式。
```

### 技巧 2：要求脱敏检查

在 AI 输出后，可以追加：

```
请检查以上内容，确保：
1. 没有真实公司名、人名
2. 金额、时间已模糊化
3. 没有可识别的具体信息
4. 如有遗漏，请指出并修正
```

### 技巧 3：优化表达

如果 AI 生成的内容不够自然，可以说：

```
请优化这段表达，要求：
1. 更像真实的个人经历分享
2. 语言更口语化、自然
3. 保留关键信息，去除冗余
```

## 常见问题

**Q: 我的经历太普通，值得分享吗？**
A: 绝对值得！项目的价值在于真实和多样。即使是"小"问题，也可能帮助很多人。

**Q: AI 脱敏可靠吗？**
A: AI 会帮你识别大部分敏感信息，但最终确认必须人工完成。请仔细检查每一条。

**Q: 我不想用 AI，可以手动写吗？**
A: 当然可以！AI 只是辅助工具，你可以直接使用 [真实故事提交指南](../../docs/01-真实故事提交指南.md) 中的模板。

**Q: 我的经历涉及法律纠纷，能分享吗？**
A: 可以，但请特别注意脱敏，避免影响正在进行或可能发生的法律程序。如有疑虑，建议咨询律师。

## 示例对话

**用户**：你好，我想分享一个经历。

**AI**：你好！我很乐意帮你整理这个真实故事。首先，请简单描述一下这是什么类型的项目？

**用户**：是一个品牌设计项目，给客户做 VI 系统。

**AI**：了解了。请告诉我：
1. 这是什么行业的客户？（如：互联网、消费品、教育等）
2. 项目预算是大概什么量级？（如：五位数、六位数）
3. 你是如何接触到这个项目的？

**用户**：客户是 [某新消费茶饮品牌]，预算 [约六位数]，通过朋友介绍认识的。

**AI**：好的，已记录。接下来请描述一下遇到的具体问题是什么？

...（继续对话）...

## 隐私提醒

⚠️ **重要**：
1. 不要在 Prompt 中透露真实敏感信息
2. AI 生成的内容必须经过人工审查
3. 最终提交前，再次检查脱敏完整性
4. 如有疑虑，宁可过度脱敏，不要冒险
