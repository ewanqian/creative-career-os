# 合同指南：修改次数限制

**明确修改边界，避免无限循环，保护你的时间和价值。**

---

## 为什么这很重要

"再改一版看看"是创意行业最令人头疼的问题之一。

如果没有明确的修改次数限制，你可能会陷入：
- 无限循环的修改，项目永远结束不了
- 投入大量额外时间，却得不到相应报酬
- 设计质量下降，因为时间被分散到太多方向
- 客户关系恶化，因为双方对"完成"的定义不一致

通过合理的修改次数约定，你可以：
- 明确项目边界，知道何时算"完成"
- 保护你的时间成本，避免无限投入
- 促使客户更认真地对待每轮反馈
- 建立专业的合作框架

---

## 两种常用方案

### 方案 A：固定次数 + 范围界定

**核心思路**：在合同中明确约定修改次数，并清晰界定什么算"修改"、什么算"新增"。

**适用场景**：
- 品牌设计项目（logo、VI、包装）
- 视觉设计项目（海报、主视觉、插画）
- UI/UX设计项目
- 任何需要多轮迭代的创意工作

**参考条款**：
```
第X条 修改与调整

1. 修改次数
   甲方有权在初稿交付后要求[X]轮修改。每轮修改甲方应提供完整、明确的修改意见。

2. 修改范围界定
   以下情形属于"修改"（不额外收费）：
   a) 在已确定的创意方向上的细节优化
   b) 色彩、字体、排版等视觉元素的调整
   c) 文案内容的微调（改动幅度不超过30%）
   d) 技术性的错误修正

   以下情形属于"新增"或"变更"（需另行协商费用）：
   a) 完全推翻已确定的创意方向重新设计
   b) 新增原合同未约定的设计内容或应用场景
   c) 文案内容重写（改动幅度超过30%）
   d) 因甲方提供的基础资料错误导致的返工
   e) 甲方内部意见不统一导致的反复修改

3. 修改流程
   a) 甲方应在收到稿件后[X]个工作日内提出修改意见
   b) 超过[X]个工作日未提出意见的，视为确认通过
   c) 每轮修改乙方应在[X]个工作日内完成并提交
   d) 修改意见应以书面形式（邮件/文档）提供，口头意见仅供参考

4. 超出修改次数的处理
   超出约定修改次数的，按以下方式处理：
   a) 第[X+1]至[X+3]轮：每轮加收合同总额的[X]%
   b) 第[X+4]轮起：每轮加收合同总额的[X]%，或视为新增项目重新报价
   c) 乙方有权拒绝不合理的修改要求

5. 终稿确认
   甲方确认终稿后，视为项目完成。此后甲方要求修改的，按新增项目处理。
```

---

### 方案 B：阶段验收 + 范围冻结

**核心思路**：将项目分为多个阶段，每个阶段验收通过后冻结，防止反复推翻。

**适用场景**：
- 复杂项目，需要分阶段推进
- 客户决策流程长，需要逐步确认
- 希望减少后期修改的项目

**参考条款**：
```
第X条 分阶段验收与范围冻结

1. 阶段划分
   本项目分为以下阶段，每阶段结束后需甲方书面确认：
   - 阶段一：概念方向（提供[X]个创意方向供选择）
   - 阶段二：深化设计（基于选定方向深化）
   - 阶段三：执行完稿（最终执行和优化）

2. 阶段确认与冻结
   a) 每阶段完成后，乙方提交阶段成果
   b) 甲方应在[X]个工作日内确认或提出修改意见
   c) 阶段一经确认，该阶段内容视为冻结
   d) 如甲方要求推翻已确认阶段重新设计，视为新增项目，需重新报价

3. 阶段内修改
   a) 每阶段内包含[X]轮修改
   b) 修改意见应在该阶段内提出，跨阶段的修改视为新增需求
   c) 阶段确认后的"小修改"（如文字错误）可在终稿时统一调整

4. 特殊情况处理
   a) 因乙方设计质量问题导致的修改：不占用修改次数
   b) 因甲方提供资料错误导致的修改：视为新增，需协商费用
   c) 因市场/政策变化需调整方向：视为新增项目
```

## 如何选择适合你的方案

**问自己三个问题**：

1. **项目复杂度如何？**
   - 简单项目 → 方案A（固定次数）
   - 复杂项目 → 方案B（阶段验收）

2. **客户决策流程如何？**
   - 决策快 → 方案A
   - 决策慢、需要多轮汇报 → 方案B

3. **你的时间成本如何？**
   - 时间充裕 → 方案A，可以多轮修改
   - 时间紧张 → 方案B，阶段冻结减少反复

## 与客户沟通的话术

### 场景：客户说"改到满意为止"

**温和但坚定的回应**：
```
感谢您的信任。关于修改，我建议我们在合同中明确修改次数和范围。

这样对我们双方都更有保障：
- 您可以更谨慎地提出修改意见，避免遗漏
- 我可以更专注地执行，确保质量

我建议的修改安排是：初稿后提供2轮修改。如果需要更多调整，我们可以按实际工作量协商额外费用。

您看这样的安排是否可以？
```

**专业且有理有据的回应**：
```
理解您希望获得满意成果的心情。从专业角度，我建议采用"阶段确认+有限修改"的模式。

原因如下：
1. **效率更高**：明确的修改轮次促使双方更认真地对待每一轮反馈
2. **质量更好**：设计师在明确边界的情况下能更专注地优化
3. **成本可控**：避免无限修改导致的项目延期和成本失控

我的建议：
- 概念阶段：提供2-3个方向供选择
- 深化阶段：2轮修改
- 完稿阶段：1轮微调

超出部分按实际工作量收费。这样既保证质量，也控制成本。
```

### 场景：客户反复修改，已超出约定次数

**你的沟通**：
```
您好，根据我们的合同约定，本项目包含[X]轮修改。目前我们已进行到第[X]轮。

我理解您希望尽善尽美的心情。为了确保项目顺利推进，我建议：

**方案A**：本轮修改后如仍需调整，我们可以按[金额/小时]的费率继续优化

**方案B**：我们可以先确定当前版本为"阶段性成果"，后续优化作为2.0版本另行规划

您倾向于哪种方案？或者我们可以电话沟通一下具体需求？
```

## 核心原则

### ✅ 要做的

1. **明确"一轮"的定义**
   - 是指提交一次修改稿，还是指完成一次修改循环
   - 在合同中明确界定

2. **保留修改记录**
   - 保存每轮修改的意见和交付记录
   - 便于后续核对和结算

3. **及时提醒次数**
   - 在接近修改次数上限时主动提醒客户
   - 避免最后才发现超出的尴尬

4. **区分"修改"和"优化"**
   - 有些客户认为"优化"不算修改，需提前界定
   - 在合同中明确哪些属于修改范围

5. **设置合理次数**
   - 根据项目复杂度设置，通常2-3轮为宜
   - 太少可能影响质量，太多容易失控

### ❌ 不要做

1. **不要模糊承诺**
   - 避免"改到满意为止"这样的表述
   - 明确次数和范围

2. **不要事后涨价**
   - 超出次数的收费标准要在合同中就明确
   - 不要临时加价让客户感觉被坑

3. **不要情绪化**
   - 即使客户反复修改，也要保持专业
   - 聚焦于解决问题，而非指责客户

4. **不要无限妥协**
   - 超出约定次数后，要有勇气提出收费
   - 你的时间也是有价值的

## 常见问题

**Q：客户说"就最后改一次"，但已经改了5次了，怎么办？**

A：温和但坚定地提醒：
```
理解您的想法。不过我们已经进行了5轮修改，超出了合同约定的[X]轮。

为了确保项目顺利完成，我建议：
- 本轮作为最后一次免费修改
- 如仍需调整，按[X]元/小时计费
- 或者我们先确定当前版本，后续优化作为2.0版本

您看哪种方式更合适？
```

**Q：客户内部意见不统一，导致反复修改，算谁的？**

A：这应该在合同中明确：
```
因甲方内部意见不统一导致的反复修改，视为新增需求，需另行协商费用。
```

实际操作中：
- 第一次出现这种情况，可以善意提醒
- 如果反复发生，需要正式提出收费

**Q：客户说"这点小改动也要算一轮？"**

A：解释"一轮"的定义：
```
根据合同约定，"一轮修改"是指：
- 甲方提出完整的修改意见清单
- 乙方根据清单完成所有修改
- 提交修改后的版本

即使是小改动，也需要经过完整的流程，确保不遗漏、不出错。

如果您有多处小改动，建议集中在一轮中提出，这样效率更高。
```
