#### 🌱 项目是什么？
这是一个**开源、可生长**的创意从业者生存指南。我们通过匿名共享真实的"战损报告"（合同陷阱、报价陷阱、沟通陷阱），共同提炼出能复用的 **"生存模组"** 。这里不提供标准答案，只提供由无数真实教训凝结而成的 **"防身工具"** 和 **"作战地图"** 。

#### 🧩 核心共建方式
我们采用"提出病例，共创药方"的模式：
1.  **提交病例**：在 [Issues](https://github.com/ewanqian/creative-career-os/issues) 区，用 `[病例]` 标签匿名描述你遇到的困境（如："客户要求无限修改，合同却没写修改次数上限"）。
2.  **诊断与提案**：社区讨论，形成策略共识。
3.  **生成模组**：将共识沉淀为可复用的"条款模组"、"话术模组"或"流程清单"，放入 [工具箱/](./toolkits/)。
4.  **版本迭代**：模组随新案例和法律法规持续更新。

### **项目哲学与设计原则**

#### **核心协作飞轮**
1.  **[病例](./docs/01-病例提交指南.md)**：一切知识的源头。指从业者遇到的真实、具体、已脱敏的困境描述（如合同陷阱、报价难题）。
2.  **共识**：加工过程。社区在公开讨论中，对病例进行分析、诊断，提炼出解决问题的策略、话术或原则。
3.  **[模组](./toolkits/)**：固化产出。将共识结构化，形成可独立使用、版本化管理的"策略单元"，存入公共工具箱。

#### **绝对红线：隐私与安全**
- **全流程脱敏**：从病例提交的第一刻起，所有可识别信息（公司名、人名、项目名、金额）必须被替换为通用标识（如`[某大型游戏公司]`、`[客户A]`、`[预算约五位数]`）。
- **非法律意见**：项目所有产出均为**同行经验分享**，不构成法律、财务或专业建议。重大决策必须咨询持证专业人士。
- **建设性质则**：聚焦于"如何解决"，禁止人身攻击与单纯的情绪宣泄。不建设无解决方案的批判。

### **AI辅助工具**

为了降低贡献门槛并提升系统智能，我们开发了 **[CCOS-Agent](./tools/ccos-agent/)** —— 一个辅助脑而非主宰，帮助用户完成从模糊抱怨到结构化病例的转换，同时严守社区的安全红线。

**核心功能**：
- **新手引导员**：通过问答引导用户描述事件、问题、诉求
- **格式助理**：自动生成符合社区标准的 Markdown 文档
- **隐私哨兵**：自动识别并替换敏感信息，确保脱敏完整性

详见：[CCOS-Agent 使用指南](./tools/ccos-agent/prompt-guide-zh.md) 和 [离线模板生成器](./tools/ccos-agent/template-generator.html)
