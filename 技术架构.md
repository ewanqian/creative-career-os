# Creative-Career-OS 技术架构

## 一、系统概述

### 核心理念
- **简单直接**：GitHub文档，无需复杂服务器
- **开源免费**：CC BY-NC-SA 4.0协议，永久免费使用
- **一人维护**：直接编辑Markdown即可
- **离线优先**：文档可下载，支持离线使用

### 系统组成
1. **工具包**：Markdown格式的实用文档（核心）
2. **项目文档**：白皮书、指南、词典等
3. **病例档案**：真实案例（待收集）
4. **GitHub托管**：文档存储、版本控制

## 四、工作流程

### 创建新工具包

```
1. 复制现有工具包作为模板
2. 修改标题和元信息
3. 填写核心内容
4. 添加话术/模板
5. 添加行动清单
6. 保存到 toolkits/ 目录
7. 更新 README.md 导航
8. 提交到GitHub
```

### 更新现有工具包

```
1. 编辑 toolkits/xxx.md
2. 更新版本号（v1.0 → v1.1）
3. 添加版本历史记录
4. 提交到GitHub
```

## 五、GitHub集成

- 所有文档使用Git版本控制

```

    ↓

    ↓

    ↓
审核合并
```

## 六、使用方式

### 方式1：直接阅读（推荐）
1. 打开GitHub仓库
2. 浏览 `toolkits/` 目录
3. 点击任意Markdown文件阅读
4. 复制需要的内容直接使用

### 方式2：下载离线使用
1. 点击右上角 **Code** → **Download ZIP**
2. 解压后浏览 `toolkits/` 目录
3. 使用任意文本编辑器打开
4. **完全离线，无需网络**

### 方式3：贡献内容
1. 阅读 `docs/01-病例提交指南.md`
2. 提交病例到 `archives/`
3. 或直接编辑工具包
4. 提交Pull Request

## 九、总结

Creative-Career-OS采用极简架构：
- **内容层**：Markdown工具包
- **托管层**：GitHub（免费、可靠、开放）

没有复杂的服务器、没有数据库、没有繁琐的工具链。

**就是简单的Markdown文档，帮助创意从业者避坑。**

这才是"有效加速"，而不是"无效加层"。
