# 病例提交指南

## 什么是"病例"？

**病例**是 Creative-Career-OS 知识体系的源头。它指的是从业者遇到的真实、具体、已脱敏的困境描述。每一个病例都是一个知识矿点，经过社区讨论和提炼，最终转化为可复用的"模组"。

## 为什么需要提交病例？

1. **帮助他人**：你的一次惨痛经历，可能成为他人避坑的明灯
2. **获得帮助**：通过社区讨论，获得多元视角的解决方案
3. **沉淀知识**：将个人经验转化为公共资产，实现价值最大化
4. **社区共建**：参与构建行业共用的"智力基础设施"

## 病例提交流程

### 第一步：准备你的故事

在提交之前，请先回忆并整理你的经历：
- 发生了什么？（事件背景）
- 问题出在哪里？（核心困境）
- 你当时如何应对？（你的行动）
- 结果如何？（最终结局）
- 如果重来，你会怎么做？（反思与建议）

### 第二步：彻底脱敏（最重要！）

**绝对红线**：所有可识别信息必须被替换！

#### 需要替换的信息类型：

| 信息类型 | 错误示例 | 正确示例 |
|:---|:---|:---|
| **公司名称** | 腾讯科技、阿里巴巴 | [某大型互联网公司]、[某知名电商平台] |
| **人名** | 张经理、李总 | [客户A]、[对接人B] |
| **项目名称** | 王者荣耀春节活动、双11主视觉 | [某游戏春节活动]、[某电商大促项目] |
| **具体金额** | 5万元、8000元 | [约五位数预算]、[约中四位数] |
| **具体时间** | 2024年3月15日 | [去年春季]、[约半年前] |
| **地点** | 北京朝阳区、上海陆家嘴 | [一线城市]、[某商业中心] |
| **作品名称** | 《星空》系列、《城市之光》 | [某系列作品]、[某主题创作] |

#### 脱敏检查清单：

- [ ] 所有公司名称已替换为通用描述
- [ ] 所有人名已替换为代号（如[客户A]）
- [ ] 所有具体金额已模糊化（如[约五位数]）
- [ ] 所有具体时间已相对化（如[去年]）
- [ ] 所有具体地点已泛化（如[一线城市]）
- [ ] 所有作品名称已匿名化
- [ ] 所有联系方式、邮箱、电话已删除
- [ ] 检查是否有组合信息可推断出具体身份

### 第三步：使用标准模板

```markdown
## 病例标题：[一句话概括核心问题]

### 基本信息
- **病例编号**：[待分配]
- **提交日期**：[YYYY-MM-DD]
- **涉及领域**：[合同/报价/沟通/版权/其他]
- **职业身份**：[自由设计师/插画师/新媒体运营/其他]
- **项目类型**：[品牌设计/UI设计/插画/视频/其他]

### 事件背景
[描述项目的来龙去脉，包括：
- 如何接触到这个项目
- 客户的基本情况（已脱敏）
- 项目的初步范围和预期
- 当时的合作模式]

### 核心困境
[清晰描述遇到的具体问题：
- 问题是什么？
- 何时发现的？
- 问题的严重程度？
- 对你造成了什么影响？]

### 你的应对
[描述你当时采取的行动：
- 第一反应是什么？
- 尝试了哪些解决方案？
- 与客户的沟通过程？
- 最终结果如何？]

### 反思与教训
[总结经验教训：
- 如果重来，你会怎么做？
- 哪些信号应该提前警惕？
- 有什么预防建议？
- 希望获得社区什么帮助？]

### 讨论标签
#[合同陷阱] #[报价问题] #[版权纠纷] #[沟通僵局] #[其他]
```

### 第四步：提交到 GitHub Issues

1. 访问 [Issues 页面](https://github.com/ewanqian/creative-career-os/issues)
2. 点击 "New issue"
3. 标题格式：`[病例] 一句话概括核心问题`
4. 将脱敏后的病例内容粘贴到正文
5. 添加适当的标签（如 `病例`、`合同`、`报价` 等）
6. 提交 Issue

## 优秀病例示例

### ✅ 优秀示例

```markdown
## 病例标题：客户以"比稿"名义征集方案后融合了多家创意

### 基本信息
- **病例编号**：#001
- **提交日期**：2026-02-02
- **涉及领域**：版权
- **职业身份**：自由品牌设计师
- **项目类型**：品牌视觉识别系统

### 事件背景
去年秋季，我通过朋友介绍接触到一个[某新消费品牌]的VI设计项目。对方表示预算充足，需要完整的品牌视觉系统，包括logo、色彩体系、应用规范等。项目周期预计2个月，预算[约六位数]。

### 核心困境
在初步沟通后，客户提出"需要看到具体方案才能确定合作"，要求我先出2-3个logo概念稿参与"内部比稿"。我提供了3个方向的初步方案后，客户表示"需要给董事会汇报"，此后便失联。两个月后，我发现该品牌发布的logo与我提供的某个方案高度相似，但设计执行有所改动，且未与我签订任何合同或支付费用。

### 你的应对
1. 第一时间联系了客户，对方表示"最终选择的是另一家公司，只是巧合相似"
2. 咨询了律师，但因为没有书面合同，维权成本极高
3. 在行业内小范围提醒同行注意该客户
4. 此后对所有新客户都坚持"先签约再出方案"的原则

### 反思与教训
- **警示信号**：客户急于看方案但对合同条款含糊其辞
- **预防建议**：任何方案展示前必须有书面协议，明确知识产权归属
- **改进方向**：希望社区能共同制定一套"比稿保护协议模板"

### 讨论标签
#版权 #合同陷阱 #比稿 #预防机制
```

### ❌ 不合格示例

```markdown
## 病例标题：被腾讯坑了

去年3月15日，我在北京朝阳区给腾讯科技做王者荣耀春节活动的设计，
项目经理张经理让我先出方案看看，预算5万。我做了3个方案后发过去，
后来他们没采用但是用了我的创意。我打电话010-12345678给张经理，
他说这是巧合。我很生气但是没办法。
```

**问题**：
- ❌ 包含具体公司名称（腾讯科技）
- ❌ 包含具体人名（张经理）
- ❌ 包含具体项目名称（王者荣耀春节活动）
- ❌ 包含具体金额（5万）
- ❌ 包含具体时间（去年3月15日）
- ❌ 包含具体地点（北京朝阳区）
- ❌ 包含联系方式（电话）
- ❌ 缺乏结构化信息
- ❌ 情绪化表达，缺乏建设性

## 常见问题 FAQ

**Q: 我的经历不够"精彩"，值得提交吗？**
A: 绝对值得！项目的价值在于真实和多样，而非戏剧化。每一个真实的困境都可能帮助他人避免同样的错误。

**Q: 脱敏后案例会不会失去参考价值？**
A: 不会。真正有价值的不是"谁"遇到了问题，而是"什么问题"以及"如何解决"。脱敏后的案例反而更具普适性。

**Q: 我可以提交正在进行的项目吗？**
A: 建议等待项目结束后再提交，这样你可以分享完整的应对过程和最终结果。

**Q: 如果我的病例涉及法律纠纷，还能提交吗？**
A: 可以，但请特别注意脱敏，避免影响正在进行或可能发生的法律程序。如有疑虑，建议咨询律师。

**Q: 提交后多久会有回应？**
A: 社区成员会在看到病例后进行讨论。通常在1-2周内会有初步讨论，但深度共识的形成可能需要更长时间。
